Bonjour, afin de pouvoir compiler les différents programmes. Il faut:
		Aller dans le dossier (versions).
		Vous aurez 3 dossiers (version1, version2, version3) et un programme (main.cs)
		qui est le programme final qui va générer les pages html(nuages de mot).

	Tout d'abord, si vous voulez compiler le fichier (main.cs), cela peut prendre un certain temps 
(environs 78pages html). Vous pouvez voir ces pages html dans le dossier (nuage de mot) et 
après, vous pouvez vous ballader dans les dossiers (periodes) ou (présidents) pour voir les résultats.
Bien entendu, pour compiler le programme, il suffit d'aller dans le terminal (linux) et tapper
mcs main.cs && mono main.exe (entrer).

	Ensuite, les 3 dossiers (version1, version2, version3), vous avez les programmes c# avec
un dossier qui contient les organigrammes pour chaque programmes. Le dossier (version3) contient
3 sous dossiers (a, b, c), donc pour 3a, 3b, 3c. 
Quand vous compilerez ces programmes, ils auront pas tous les memes procédés, c'est-à-dire que les programmes
évoluent au fur et à mesure. Dans la version a et b, il faut rentrer le dosier (président) à la main et la (date).
Pour la version3, tous va commencer à etre automatisé. C'est à partir de la version 3c que cela va etre 100% automatisé.
Petit point important, vous pouvez voir les résultats de compilation pour la version 3c dans le dossier (résultatCompilation)
qui se trouve au début.

	Enfin, les compilations peuvent prendre un certain temps, surtout à partir de la version 3, car il fait chaque date
et chaque période. Donc pour voir si il y a pas une boucle infini, vous pourrez voir sur le terminal, un indicateur quand 1 discours a été traité.
